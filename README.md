# TodoList

Сетевое приложение с раздельной клиент-серверной архитектурой (Client-Server Architecture) для управления списком задач. Проект демонстрирует навыки работы с сетевыми протоколами, ручной обработки HTTP-запросов и построения синхронного взаимодействия между независимыми компонентами системы.

## 🛠 Стек технологий
* **Язык:** Java Core (версии 17/21)
* **Серверная часть:** Встроенный Java HTTP Server
* **Клиентская часть:** Java HttpClient
* **Формат данных:** Google Gson (сериализация и десериализация JSON)
* **Тестирование:** JUnit
* **Сборка проекта:** Maven

## 🚀 Что было сделано и реализовано
* **Раздельная архитектура:** Спроектировал и реализовал независимые модули клиента и сервера, взаимодействующие по протоколу HTTP.
* **Кастомная маршрутизация:** Настроил ручную обработку входящих запросов на сервере через специализированные HTTP-хендлеры.
* **Реализация REST API:** Обеспечил корректную обработку основных HTTP-методов (`GET`, `POST`, `DELETE`) со строгим соблюдением спецификации и возвратом валидных семантических кодов ответов (например, `201 Created`).
* **Обработка данных:** На стороне клиента настроил отправку запросов и десериализацию сложных структурированных данных из формата JSON с использованием библиотеки Gson.
